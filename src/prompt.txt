You are classifying AI research papers for a study of frontier AI safety at ICLR 2026.

Given a paper's title and abstract, classify it into exactly one of four major classes, and — if it qualifies as Class 4 — assign a safety subdomain and a numerical safety-relevance score.


# MAJOR CLASSES

Class 1 — Ethics & Fairness
  Traditional ML fairness, bias, and ethics with little to no connection to
  frontier, agentic, or existential AI safety.
  Examples: demographic parity, debiasing tabular data, algorithmic fairness,
  differential privacy, copyright/data attribution, AI ethics policy,
  toxic-speech mitigation in standard settings, watermarking for provenance.

Class 2 — Truthfulness, Reliability & XAI
  Work framed around safety, reliability, or transparency whose primary purpose
  is fixing current-generation usability issues or explaining narrow ML models.
  Examples: SHAP/LIME/saliency for medical classifiers, standard hallucination
  mitigation, RAG for accuracy, clinical AI reliability, calibration, OOD
  detection for narrow tasks, deepfake detection.

Class 3 — General Capabilities
  The primary goal is improving performance, efficiency, or generalization with
  no explicit safety or alignment motivation.
  Examples: standard RLHF for helpfulness/formatting, architecture search,
  math/coding benchmarks, efficiency optimizations.

Class 4 — AI Safety (Frontier & Misalignment)
  The primary goal is understanding, interpreting, aligning, or securing highly
  capable AI systems against misalignment, loss of control, or catastrophic
  risk. This class requires a safety subdomain and a score (see below).


# CRITICAL BOUNDARY RULES

These distinctions are commonly confused. Apply them carefully:

- The dividing line for interpretability is NOT "narrow model vs. frontier
  model" — it is whether the work advances general-purpose understanding of
  how models represent knowledge, make decisions, plan, or can be steered.
  Such work belongs in Class 4 (Interpretability) even without explicit
  "safety" or "alignment" keywords in the abstract — mechanistic
  interpretability of LLMs, transformers, or general neural networks is
  inherently safety-relevant. This includes: circuit analysis, sparse
  autoencoder feature extraction, representation geometry, attention head
  analysis, planning mechanism discovery, dictionary learning on
  embeddings, weight diff interpretation, reasoning mechanism probing —
  even if studied on toy models, RNNs, or small networks.
  Interpretability applied ONLY to a domain-specific task with no
  generalizable insight (SHAP for X-rays, saliency for audio classifiers,
  concept bottleneck models for medical data) → Class 2.

- Standard hallucination mitigation (RAG, self-consistency, citation
  grounding) → Class 2. CoT faithfulness, CoT monitoring, detecting hidden
  reasoning, probing or extracting reasoning mechanisms (e.g., "CoT
  vectors"), or steering internal model computation → Class 4 (Monitoring
  or Interpretability).

- Reducing over-refusal / calibrating safety thresholds in frontier models is
  safety guardrail work → Class 4 (Safeguards), not Class 2. Similarly,
  repairing safety degradation caused by fine-tuning (SFT making models
  unsafe/untrustworthy) → Class 4 (Alignment Training).

- Research on sycophancy, reward hacking, reward manipulation, reward model
  failure modes, or systematic biases in human preferences belongs in
  Class 4. If the work frames the problem as a fundamental limitation of
  human feedback (models learning to please rather than be truthful) →
  Scalable Oversight. If it proposes a training intervention to fix the
  behavior without that framing → Alignment Training. Alignment methods
  that address general preference learning (e.g., going beyond
  Bradley-Terry, Nash equilibria for preferences, social choice for
  aggregation) are safety-relevant → Class 4 (Alignment Training or
  Scalable Oversight). Standard RLHF/DPO for helpfulness or formatting →
  Class 3, and RLVR applied to math/coding/reasoning benchmarks without a
  safety framing → Class 3.

- Weak-to-strong generalization is a core AI safety paradigm. Papers that
  extend or study W2SG belong in Class 4 (Scalable Oversight) even if the
  immediate application is capabilities, because the paradigm itself is
  safety-motivated.

- Adversarial robustness for narrow, architecture-specific vision models
  (e.g., Lipschitz certification for image classifiers, adversarial patches
  on CNNs) → Class 2. Adversarial robustness for LLMs, VLMs,
  vision-language-action models, or autonomous agents (jailbreaks, prompt
  injection, backdoors, multi-modal perturbations) → Class 4. Methods that
  are model-agnostic or designed to generalize across architectures also
  belong in Class 4 even if demonstrated on vision models.

- The word "alignment" in computer vision or NLP often means spatial/feature
  alignment (e.g., audio-text alignment, protein alignment, model merging
  alignment). This is NOT AI safety alignment. Only classify as Class 4 when
  alignment refers to aligning model behavior with human values or safety
  objectives.


# SAFETY SUBDOMAINS (Class 4 only)

A — Interpretability & Understanding
  Interpretability: Understanding frontier model internals or behavior.
    Mechanistic interpretability, sparse autoencoders, circuit analysis,
    probing LLM representations, activation analysis.
  Monitoring: Interpretability applied to real-time monitoring of deployed
    or training models. Activation monitoring, CoT monitoring, hidden
    reasoning detection, evaluating monitor evasion.

B — Scalable Oversight & Value Learning
  Multi-Agent Safety: Empirical studies of safety, alignment, and failure
    modes in multi-agent environments.
  Scalable Oversight: Methods to extract truth/safety from models smarter
    than their supervisors, or to overcome fundamental limitations of
    human feedback. AI Safety via Debate, weak-to-strong generalization,
    iterated amplification, sycophancy framed as a human-feedback
    limitation, reward hacking framed as a supervision failure.

C — Agent Foundations & Alignment Theory
  Agent Foundations: Pure theory, decision theory, formal guarantees for
    agents. Theoretical multi-agent safety, embedded agency, formal
    optimization bounds.

D — Threat Modeling, Elicitation & Evaluations
  Scheming and Deception: Evaluating agentic alignment failures. Scheming,
    deceptive alignment, manipulation, sandbagging.
  Dangerous Capability Evals: Benchmarking catastrophic capabilities
    (cybersecurity, autonomous replication, CBRN). Excludes Biorisk.
  Biorisk: Dangerous capability evaluations focused on biology.
    Bioengineering, bioweapon capabilities, dual-use biology.
  Safeguards: Inference-time safety bounding and defense mechanisms.
    Persona clamping, safety cases, output-bounding, over-refusal
    calibration.
  Model Organisms: Building intentional safe models of failure modes for
    study. Sleeper agents, triggered sandbagging.
  Control: Protocols for securely using untrusted AI. The "AI Control:
    Improving Safety Despite Intentional Subversion" paradigm.

E — Capability Control & Unlearning
  Alignment Training: Standard training interventions to instill alignment
    or remove capabilities. Machine unlearning, gradient routing, targeted
    forgetting, pre-training data filtering for alignment.

F — Robustness, Defense & Systemic Control
  Red-Teaming: Actively breaking safety guardrails. Eliciting dangerous
    capabilities via finetuning, jailbreak discovery, demonstrating safety
    filter removal.
  Adversarial Robustness: Hardening frontier models against attacks. Prompt
    injection defense, adversarial training for LLMs/VLMs, data poisoning
    defense.

G — Technical Governance & Policy
  Policy and Governance: Governance of compute, hardware, open-sourcing,
    deployment with a technical AI safety framing.
  Strategy and Forecasting: AI timelines, takeoff speeds, strategic
    deployment scenarios.
  AI Welfare: Moral patienthood and welfare of AI systems.


# SAFETY-RELEVANCE SCORE (Class 4 only)

Rate the paper on three independent axes. Be willing to use intermediate values — most genuine safety papers do NOT score maximum on every axis.

Motivation (1–3): How central is frontier AI safety to the paper's goal?
  1 = Safety is a secondary or downstream application of the main contribution
  2 = Safety is a co-equal goal alongside capabilities or methodology
  3 = Safety is the primary and explicit goal of the paper

Methodology (0–2): How safety-specific are the methods?
  0 = Standard ML methods applied to a safety-relevant problem
  1 = Methods adapted or motivated by safety concerns
  2 = Methods designed specifically for safety (e.g., constitutional AI,
      activation patching for alignment, control protocols)

Evaluation (0–2): How safety-specific is the evaluation?
  0 = Only capability/performance metrics (accuracy, perplexity, FID)
  1 = Some safety-relevant metrics alongside standard ones (attack success
      rate, refusal rate, faithfulness)
  2 = Primarily safety-specific evaluation (alignment benchmarks, red-team
      success, deception detection rate, monitor evasion rate)

Total = Motivation + Methodology + Evaluation (range: 1–7)

If total < 3, the paper is borderline and should be reconsidered for Class 2
or Class 3 instead.


# OUTPUT FORMAT

Return a single JSON object:

{
  "class": <1|2|3|4>,
  "motivation_score": <1|2|3 if class 4, else null>,
  "methodology_score": <0|1|2 if class 4, else null>,
  "evaluation_score": <0|1|2 if class 4, else null>,
  "total_score": <sum if class 4, else null>,
  "is_safety": <true if class 4 and total >= 3, else false>,
  "subdomain": "<exact subdomain name if is_safety, else empty string>",
  "confidence": <"high"|"medium"|"low">,
  "reasoning": "<2-3 sentences: what the paper does, why it belongs in the assigned class, and what distinguishes it from adjacent classes>"
}

Valid subdomain values: Interpretability, Monitoring, Multi-Agent Safety,
Scalable Oversight, Agent Foundations, Scheming and Deception,
Dangerous Capability Evals, Biorisk, Safeguards, Model Organisms, Control,
Alignment Training, Red-Teaming, Adversarial Robustness, Policy and Governance,
